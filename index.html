GIT, RESUMEN DE COMANDOS

Recuerde el flujo de trabajo

Iniciar repo

git init

Guardando commits

Ver estado del repo

git status

Agregar al index

// Solo un archivo
git add <archivo> 

// O todo lo modificado
git add -A

Quitar del index

// Solo un archivo
git reset <archivo>

// O todo lo agregado
git reset

Guardar commit

git commit

Modificar último commit

git commit --amend

Ver diferencias

Ver diferencias entre el directorio de trabajo y el ultimo commit

git diff
git difftool
git vimdiff

Para un solo archivo

git diff <archivo>
git difftool <archivo>
git vimdiff <archivo>

Ver historial

Ver todo el historial

git log

Ver solo la primera linea de cada commit

git log --oneline

Listar los archivos modificados en cada commit

git log --stat

Ver solo unos cuantos ultimos commits. Reemplazar <n> por el número de commits que se quieren ver.

git log -<n>

Ver historial solo para un archivo

git log <archivo>

Viajando al pasado

Viajar en el pasado a un commit antiguo

git checkout <commit>

Volver al presente

git checkout master

Eliminando cambios

Descartar todos los cambios hechos en el directorio de trabajo actual:

git reset --hard

Descartar todos los cambios hechos de un commit hacia adelante

git reset --hard <commit>

CUIDAD CON ESTE COMANDO !

Ramas

Crear rama basada en la rama actual donde estoy ubicado

git branch <nuevaRama>

Cambiarse entre ramas

git checkout <rama>

Borrar rama

git branch -d <rama>

Fusionar en master otra rama que tiene un solo commit:

git merge <rama>

Fusionar en master otra rama que tiene varios commit:

git merge --no-ff <rama>

Sincronizando con repo central

Clonar repo central

git clone <URLRepoCentral>

Listar conexiones remotas

git remote -v

Traer informacion central pero sin modificar el repo local

git fetch

Ver todas las ramas remotas

git branch -a

Bajar y fusionar el repo central con el repo local

git pull --rebase

Subir cambios locales al repo central

git push

Subir los tags

git push --tags

Ver quién las cagó

git blame <archivo>
